# DataPilot 화면 설계서

> 프로토타입 화면의 **기능/상호작용/데이터 바인딩 명세**.
> 시각 레퍼런스는 `docs/wireframe.html` 참조.
> Figma 작업 시 본 문서의 넘버링 체계를 어노테이션으로 그대로 사용.

---

## 개요

| 항목 | 내용 |
|---|---|
| 화면 수 | 3장 (시작 / 분석 진행 / 리포트) |
| 플로우 | 시작 → 분석 진행 → 리포트 (선형) |
| 역방향 | 리포트 → 시작 (재분석) |

### 화면 전환 플로우

```
[화면 1: 시작]
    │ [분석 시작] 클릭
    ▼
[화면 2: 분석 진행]
    │ 6개 에이전트 순차 완료
    ▼
[화면 3: 리포트]
    │ [새 분석 시작] 클릭
    ▼
[화면 1: 시작]
```

### 예외 플로우

```
[화면 2: 분석 진행]
    │ 에이전트 실패
    ▼
[화면 2-E: 오류]
    │ [다시 시도] 또는 [처음으로]
    ▼
[화면 2: 분석 진행] 또는 [화면 1: 시작]
```

## 화면 1: 시작

### 1.0 화면 개요
- **목적**: 분석 대상 게임과 기간을 선택해 분석을 시작한다.
- **진입 경로**: 앱 접속 시 기본 화면 / 리포트 화면에서 "새 분석 시작" 클릭
- **다음 화면**: 화면 2 (분석 진행)

### 1.1 헤더
- **요소**: 제품명 `DataPilot` (H1) + 서브 카피 한 줄
- **서브 카피**: "운영 지표에서 병목을 찾고, 원인을 분석하고, 액션을 제안합니다."
- **동작**: 정적 표시, 인터랙션 없음

### 1.2 게임 선택 드롭다운
- **요소**: 라벨 "게임 선택" + 드롭다운
- **데이터**: 사용자가 담당하는 게임 목록
  - 프로토타입에서는 `Pizza Ready` 1개만 표시
  - 실제 운영에서는 사용자의 담당 게임 여러 개
- **기본값**: 첫 번째 게임 자동 선택
- **동작**: 클릭 → 게임 목록 펼침 → 선택 시 해당 게임의 스키마와 데이터 소스 로드
- **검증**: 필수 선택 (기본값이 있으므로 실제로는 빈 값 불가)

### 1.3 분석 기간 라디오
- **요소**: 라벨 "분석 기간" + 라디오 버튼 3개
- **옵션**: `최근 7일` / `최근 14일` / `최근 30일`
- **기본값**: `최근 30일` (가장 많은 데이터로 정확도 높임)
- **동작**:
  - 하나만 선택 가능 (단일 선택)
  - 선택된 옵션은 강조 스타일 (테두리 `#e74c3c`, 배경 `#fef2f2`)
- **검증**: 하나는 반드시 선택됨 (기본값 보장)

### 1.4 분석 시작 버튼
- **요소**: 풀 너비 primary 버튼 "분석 시작"
- **동작**:
  - 클릭 → 1.2와 1.3 값을 payload로 만들어 파이프라인 실행
  - 즉시 화면 2로 전환
- **상태**:
  - `default`: 활성화
  - `loading`: 클릭 직후 짧은 시간, 회색 + 스피너
  - `disabled`: 드롭다운 미선택 시 (프로토타입에서는 발생 X)

### 1.5 에러 처리 (엣지 케이스)
- 게임 목록 로드 실패 시: 드롭다운 자리에 "게임 목록을 불러올 수 없습니다. 새로고침 해주세요." 안내
- 분석 시작 실패 시: 버튼 아래 에러 메시지 표시 후 버튼 재활성화

## 화면 2: 분석 진행

### 2.0 화면 개요
- **목적**: 6개 에이전트가 순차적으로 실행되는 동안 진행 상태를 실시간 표시.
- **진입 경로**: 화면 1에서 "분석 시작" 클릭
- **다음 화면**: 화면 3 (모든 에이전트 완료 시 자동 전환)
- **소요 시간 예상**: 약 30초 ~ 2분 (Claude API 호출 시간의 합)

### 2.1 화면 제목
- **요소**: H2 "{게임명} ({분석기간}) 분석 중..."
- **데이터 바인딩**:
  - `{게임명}` ← 화면 1의 1.2에서 선택한 값
  - `{분석기간}` ← 화면 1의 1.3에서 선택한 값
- **동작**: 정적 표시

### 2.2 에이전트 진행 스텝 리스트
- **요소**: 6개 스텝 세로 나열 (`step-done` / `step-active` / `step-wait`)
- **스텝 구성** (순서 고정):
  1. 병목 감지
  2. 세그먼트 분석
  3. 가설 생성
  4. 데이터 검증
  5. 원인 추론
  6. 액션 추천

- **각 스텝의 구성**:
  - 번호 아이콘 (원형)
  - 에이전트 명칭 (한글)
  - 상태 텍스트 (동적)

- **상태 전환 규칙**:
  | 상태 | 아이콘 배경 | 상태 텍스트 예시 |
  |---|---|---|
  | `wait` | 회색 `#eee` | "대기" |
  | `active` | 빨강 `#e74c3c` + 흰 텍스트 | "분석 중..." |
  | `done` | 초록 `#d4edda` + 초록 텍스트 | "완료 — {요약}" |

- **완료 시 요약 규칙**:
  - 병목 감지 완료: "완료 — 이상 지표 N개 발견" (N = 탐지된 이상 수)
  - 세그먼트 분석 완료: "완료 — {집중 세그먼트}" (예: "Android 집중")
  - 가설 생성 완료: "완료 — 가설 N개 발산"
  - 데이터 검증 완료: "완료 — 확인 N / 기각 N / 미검증 N"
  - 원인 추론 완료: "완료" 또는 "완료 — 원인 불명" (supported 0개인 경우)
  - 액션 추천 완료: "완료 — 액션 N개 제안"

- **동작**:
  - 한 번에 하나의 스텝만 `active`
  - 이전 스텝이 `done`이 되면 다음 스텝이 `active`로 전환 (순차)
  - 중간 실패 시 해당 스텝은 `error` 상태 (화면 2-E로 분기)

### 2.3 경과 시간 표시
- **요소**: 스텝 리스트 하단 우측 정렬, 작은 회색 텍스트
- **포맷**: "경과 시간: {분}분 {초}초"
- **동작**:
  - 화면 2 진입 시 타이머 시작
  - 1초마다 갱신
  - 화면 3 전환 시 정지

### 2.4 에러 상태 (화면 2-E)
- **발생 조건**: 에이전트 실행 중 예외 발생 (API 에러, 타임아웃, 검증 실패 등)
- **표시 변경**:
  - 실패한 스텝의 아이콘을 빨강 X로 변경
  - 상태 텍스트: "실패 — {에러 메시지}"
  - 경과 시간 정지
- **액션 버튼 노출** (스텝 리스트 하단):
  - `[다시 시도]` — 실패 지점부터 재실행
  - `[처음으로]` — 화면 1로 복귀

## 화면 3: 리포트

### 3.0 화면 개요
- **목적**: 분석 결과를 이상 지표별로 요약 + 상세 카드로 제공.
- **진입 경로**: 화면 2에서 모든 에이전트 완료 시 자동
- **구조**: 상단 요약 (이상 지표 카드 N개) + 하단 상세 (선택된 이상 지표의 5카드)
- **스크롤**: 상단 요약 고정은 하지 않음, 페이지 전체 스크롤

### 3.1 화면 제목
- **요소**: H2 "분석 완료 — {게임명} ({분석기간})"
- **데이터 바인딩**: 화면 1의 선택값 그대로

### 3.2 이상 지표 요약 카드 (상단)
- **요소**: 카로셀 형태의 가로 카드 N개 (이상 지표 수만큼)
- **데이터 소스**: `① 병목 감지` 에이전트의 `anomalies[]` 출력
- **카드 1개 구성**:
  - 지표명 (작은 회색 텍스트)
  - 변화 표시 (큰 빨강 텍스트)
    - 상대 변화율일 때: `-11%`
    - 절대값 변화일 때: `28% → 24%` 또는 `98% → 92%`
  - 심각도 배지 (`HIGH` / `MEDIUM` / `LOW`)

- **변화 표시 규칙** (Bottleneck Detector가 판단해서 넘겨줌):
  - 매출·비용 등 금액/카운트 → 상대 변화율 (`-11%`)
  - 리텐션·성공률 등 비율 지표 → before→after (`28% → 24%`)

- **선택 상태**:
  - 카드 중 하나는 항상 `active` (기본: 첫 번째)
  - `active` 카드는 테두리 빨강 + 배경 `#fef7f7`
  - `active` 카드에 대응하는 상세 5카드가 하단에 표시됨

- **상호작용**:
  - 카드 클릭 → `active` 전환 → 하단 상세 5카드가 해당 이상 지표로 갱신
  - 키보드 ←/→ 키로 카드 전환 (선택 사항)

- **N개 처리**:
  - N ≤ 4: 가로 한 줄로 나열
  - N > 4: 2줄로 wrap (프로토타입에서는 N=3 고정)
  - N = 0: "이상 지표가 감지되지 않았습니다" 안내 + 상세 영역 숨김

### 3.3 구분선
- 이상 지표 요약과 상세 카드 사이 가로 구분선 (`#e0e0e0`)

### 3.4 상세 카드 1 — 이상 지표 요약
- **데이터 소스**: `① 병목 감지` 에이전트의 현재 선택된 anomaly 상세
- **카드 헤더**:
  - 좌: 카드 제목 "이상 지표 요약"
  - 우: 에이전트 라벨 "① 병목 감지" (작은 회색)
- **카드 본문**: 라벨-값 세로 분리 리스트
  - **지표**: 한글명 + 영문명 (예: "인앱결제 매출 (revenue)")
  - **변화율**: 절대/상대 + 비교 구간 (예: "-11.1% (직전 4일 평균 ₩351,200 → 최근 4일 평균 ₩312,400)")
  - **심각도**: `HIGH` / `MEDIUM` / `LOW`
  - **판정 근거**: 장문 텍스트
    - 형식: "시점 명시 + 변화 패턴 + 다른 지표와의 대조"
    - 예: "D-3(3/28) Android 배포 이후 매출이 지속 하락. 정상 변동 범위(±5%)를 초과하며 4일 연속 하향 추세. 같은 기간 DAU·세션 수는 정상 → 유저 감소가 아닌 결제 단가 문제로 추정."
- **동작**: 정적 표시

### 3.5 상세 카드 2 — 세그먼트 분해
- **데이터 소스**: `② 세그먼트 분석` 에이전트의 현재 이상 지표 분해 결과
- **카드 헤더**:
  - 좌: "세그먼트 분해"
  - 우: "② 세그먼트 분석"
- **카드 본문**:
  - **요약 제목** (빨강 H3): "{지표} 감소가 {집중 세그먼트}에 집중 ({변화율}, 나머지는 {정상 수치})"
    - 예: "매출 감소가 Android에 집중 (-18%, iOS는 +0.2%로 정상)"
  - **차원별 바 차트**:
    - 제목: "{차원명}별 {지표} 변화율"
    - 각 세그먼트 1줄:
      - 라벨 (우측 정렬, 80px)
      - 바 (길이 = 변화율 절대값 비례, 색상 = 음수 빨강 / 양수 초록)
      - 수치 (색상 규칙 동일)
- **집중 세그먼트 없을 때 (분산 패턴)**:
  - 제목: "{지표} 감소가 특정 세그먼트에 집중되지 않음 (전반적 하락)"
  - 바 차트는 그대로 표시
- **동작**: 정적 표시

### 3.6 상세 카드 3 — 가설과 검증
- **데이터 소스**: `③ 가설 생성` + `④ 데이터 검증` 에이전트 출력
- **카드 헤더**:
  - 좌: "가설과 검증"
  - 우: "③ 가설 생성 + ④ 데이터 검증"
- **카드 본문**: 가설 리스트 (세로 나열, 구분선)
- **가설 1개 구성**:
  - 좌측: 상태 배지 (`확인됨` / `기각됨` / `미검증`) — 고정 너비 56px
  - 우측 상단: 가설 텍스트 (13px, weight 500)
  - 우측 하단: 근거 텍스트 (11px, 회색)
    - `확인됨` 가설은 근거를 약간 진한 색 (`#555`)으로 강조
    - `기각됨` / `미검증`은 더 연한 회색 (`#888`)
- **정렬 순서**: 확인됨 → 기각됨 → 미검증
- **빈 상태**:
  - 가설 0개: "가설이 생성되지 않았습니다" 안내
- **동작**: 정적 표시 (향후 가설 클릭 시 증거 데이터 상세 펼침 가능 — Phase 2+)

### 3.7 상세 카드 4 — 근본 원인
- **데이터 소스**: `⑤ 원인 추론` 에이전트의 `root_cause` 출력
- **카드 헤더**:
  - 좌: "근본 원인"
  - 우: "⑤ 원인 추론"
- **카드 본문**: 인과 체인 (세로 단계 나열) + 결론 박스 + 추가 검토 박스

#### 3.7.1 인과 체인
- **구성**: 단계 1 → 2 → 3 (supported 가설 기반, AI가 구성)
- **한 단계**:
  - 좌: 단계 번호 원형 배지 (빨강 배경)
  - 우 상단: 단계 텍스트 (13px weight 500)
  - 우 하단: 근거 텍스트 (11px 회색, "근거: {SQL 설명 또는 테이블 출처}")
- **단계 간 연결**: 화살표 아이콘 (↓, 회색)

#### 3.7.2 결론 박스
- **요소**: 단계 리스트 하단, 빨강 좌측 보더 박스
- **텍스트**: "**결론:** {한 줄 요약}"
- **예**: "결론: Android 배포의 UI 변경이 프리미엄 상품 노출을 줄여 전사 매출 감소로 이어짐"

#### 3.7.3 추가 검토 박스 (선택적)
- **표시 조건**: `additional_investigation[]`이 비어있지 않을 때만 노출
- **요소**: 점선 테두리 박스, 회색 배경
- **제목**: "추가 검토 필요"
- **내용**: 미검증 가설 + 필요 데이터 리스트
  - 예: "- Android 크래시율 증가 여부 (크래시 로그 필요)"

#### 3.7.4 원인 불명 케이스 (supported 0개)
- 3.7.1 인과 체인 대신 안내 박스 표시:
  - "원인 불명 — 검증된 가설이 없어 인과 체인을 구성할 수 없습니다"
- 3.7.2 결론 박스: "결론: 현재 연결된 데이터로는 근본 원인을 단정할 수 없습니다"
- 3.7.3 추가 검토 박스: 모든 미검증 가설 나열 (**반드시 노출**)

### 3.8 상세 카드 5 — 추천 액션
- **데이터 소스**: `⑥ 액션 추천` 에이전트의 `actions[]` 출력
- **카드 헤더**:
  - 좌: "추천 액션"
  - 우: "⑥ 액션 추천"
- **카드 본문**: 액션 리스트 (세로 나열, 구분선)
- **액션 1개 구성**:
  - 좌: 우선순위 배지 (고정 너비 60px 중앙 정렬)
    - `긴급` — 빨강 배경 + 흰 텍스트
    - `단기` — 노랑 배경 + 갈색 텍스트
    - `중기` — 회색 배경 + 짙은 회색 텍스트
  - 우 상단: 액션 제목 (13px weight 600)
  - 우 하단: 메타 정보 2줄
    - "효과: {예상 효과}"
    - "리소스: {필요 리소스}"
- **정렬 순서**: 긴급 → 단기 → 중기
- **원인 불명 케이스**:
  - 액션이 "추가 데이터 수집" 중심으로 변경됨
  - 예: "Android 크래시 로그 수집 파이프라인 연결"
- **빈 상태**:
  - 액션 0개: "제안할 액션이 없습니다 — 원인 불명 상태" (거의 발생하지 않음)

### 3.9 하단 액션 영역 (선택 사항)
- **표시 조건**: 상세 5카드 아래

## 상태 변화 및 엣지 케이스 정리

### 상태 매트릭스
| 화면 | 상태 | 조건 | 표시 |
|---|---|---|---|
| 1 | default | 초기 진입 | 드롭다운 + 라디오 + 버튼 |
| 1 | loading | 게임 목록 로드 중 | 드롭다운 자리에 스피너 |
| 1 | error | 게임 목록 실패 | 에러 안내 + 새로고침 |
| 2 | running | 에이전트 실행 중 | 진행 스텝 + 경과 시간 |
| 2 | error | 에이전트 실패 | 실패 스텝 + [다시 시도] |
| 3 | default | 이상 N개 탐지됨 | 요약 카드 + 상세 카드 |
| 3 | empty | 이상 0개 | "정상 — 이상 지표 없음" 안내 |
| 3 | unknown | supported 0개 | 원인 불명 + 추가 검토 중심 |

### 주요 엣지 케이스
1. **이상 지표가 0개**: 화면 3에서 "모든 지표 정상" 안내 + 상세 카드 숨김
2. **supported 가설이 0개** (원인 불명): 3.7.4 참조
3. **unverified만 있고 supported/rejected 없음**: 3.6에서 미검증만 나열, 3.7은 원인 불명 처리
4. **에이전트 중간 실패**: 화면 2-E로 전환, 부분 결과는 버리고 재실행
5. **분석 기간 내 데이터 부족**: 병목 감지 단계에서 "데이터 부족" 응답 → 화면 3 empty 상태
